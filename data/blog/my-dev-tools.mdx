Como programadores solemos pasar mucho tiempo delante de nuestras computadoras. Utilizando las herramientas correctas podemos mejorar mucho la experiencia y tiempos de desarrollo de nuestras aplicaciones.

El deseo de convertirse en desarrolladores más proficientes empieza mejorando nuestras habilidades como programadores en sí, ya sea reforzando nuestras bases, aprendiendo paradigmas o tecnologías nuevas, y sobre todo implementando buenas prácticas y mejorando la calidad de nuestro código.

Sin embargo, para aprovechar nuestro máximo pontencial durante todas las horas que estamos sentados frente a nuestro escritorio leyendo y escribiendo código, es necesario también mejorar nuestro entorno y las herramientas que utilizamos. Así como por ejemplo sería el hacer ejercicio físico; si descanzamos las horas debidas y mantenemos un estilo de vida mas saludable con una buena alimentación, mejorá nuestro rendimiento, y sacaremos máximo provecho a todas esas horas en el gimnasio.

Una de las formas más sencillas para optimizar nuestros entornos de desarrollo es personalizarlo a nuestro gusto para adaptarlo a nuestras necesidades. A menor escala esto se refleja utilizando atajos en el teclado para acceder a las funciones que mas utilizamos como el debugger o el botón para compilar y correr nuestro código, pero en dependencia de los programas que utilicemos, podremos customizar aún más el comportamiento de dichas herramientas y realmente hacer el entorno de desarrollo nuestro.

En este artículo cubriré algunas personalizaciones que suelo aplicar a mis entornos de desarrollo y algunos de los programas que utilizo para mejorar mi eficiencia en mi día a día.

## Editor de Código

Como programadores nuestra herramienta más importante es siempre un editor de código en el que podamos trabajar de una manera cómoda con los lenguajes que estemos utilizando en el momento.

Existen muchos editores de código diseñados exclusivamente para cumplir esa misma función, editar texto, como lo son por ejemplo [Vim](https://www.vim.org/) o [Emacs](https://www.gnu.org/software/emacs/), e incluso [Sublime Text](https://www.sublimetext.com/). Por otra parte, para muchos lenguajes también disponemos de entornos de desarrollo con una gran cantidad de herramientas integradas (conocidos como IDEs), que generalmente incluyen todo lo que podemos llegar a necesitar, todo a través de una sola aplicación, es por esto que los IDE suelen ser específicos para un lenguaje o stack, como es el ejemplo de [PyCharm](https://www.jetbrains.com/pycharm/) o [Visual Studio](https://visualstudio.microsoft.com/).

A lo largo de todo el tiempo que llevo programando, he probado diferentes tipos de editores para diferentes lenguajes. El primero de ellos fue [CodeBlocks](https://www.codeblocks.org/) cuando recién empezaba a programar y no sabía que era un editor. Rápidamente transicioné a Visual Studio para programar en C++, pero cuando me inicié con el desarrollo web dejé de utilizar IDEs en favor de herramientas más ligeras y rápidas como Sublime Text.

Tras cierto tiempo empecé a utilizar el editor más popular entre los desarrolladores web, Visual Studio Code, debido al balance entre su facilidad de uso y la cantidad de herramientas que trae integradas. Al poco tiempo de utilizar VSCode inicié a personalizarlo a mi gusto, con la ayuda de las diferentes extensiones que se le pueden instalar facilmente, y de la excelente API que este provee para poder customizar su interfaz.

La forma más rápida para cambiar las estética de VSCode es utilizando uno de las múltiples extensiones que otros desarrolladores han publicado en la tienda de extensiones de VSCode. Los temas disponibles varían en cuanto a paletas de colores, y colorado de sintaxis. Sin embargo para controlar totalmente cómo se ven cada uno de los componentes que conforman la interfaz de VSCode, debemos referirnos al archivo de configuración del editor para detallar qué colores queremos que utilice cada uno de los elementos que queremos modificar. Para hace esto debemos abrir nuestro archivo de configuración.

<Callout>
	Para esto podemos presionar <kbd>Ctrl+Shift+P</kbd> (en Windows) y luego
	buscar "<u>settings json</u>" en la paleta de comandos. Debemos seleccionar
	la opción que solo diga "<u>Open Settings (JSON)</u>", ya que también
	podemos configurar los valores por defecto o por workspace.
</Callout>

Este archivo debe contener un objeto con nuestra configuración. Para agregar o modifiar nuestro tema, debemos crear (si no existe ya) una llave con el nombre `workbench.colorCustomizations` que a su vez es otro objeto en donde podemos ir ingresando cada una de las llaves correspondientes al elemento cuyo color vamos a modificar, por ejemplo:

```json:settings.json
{
	"workbench.colorCustomizations": {
		"activityBar.background": "#7b6781",
		"editor.background": "#3d3241",
		"tab.activeBackground": "#57495c",
        "titleBar.activeBackground": "#3d3241"
	}
}
```

Los nombres de todos los elementos que componen la interfaz de VSCode estan disponibles en la [documentación oficial](https://code.visualstudio.com/api/references/theme-color). Mi archivo de configuración está disponible en [GitHub](https://github.com/itscarlosecp/.dotfiles/blob/main/vscode/settings.json). También podemos configurar nuestra fuente (yo utilizo [JetBrains Mono](https://www.jetbrains.com/lp/mono/)), nuestro tipo y tamaño de indentado, la longitud máxima de cada línea, etc.

También utilizo extensiones para mejorar estéticamente el código en sí. Entre estas está [Prettier](https://prettier.io/), una de las más populares para ciertos lenguajes. Esta extensión nos permite formatear el código automáticamente, y podemos establecer una serie de reglas que queremos que aplique sobre el código. Estas condiciones también las podemos configurar desde nuestro archivo de configuración de VSCode, de la siguiente manera:

```json:settings.json
{
	"prettier.tabWidth": 4,
	"prettier.jsxSingleQuote": true,
	"prettier.singleQuote": true,
	"prettier.useTabs": true,
	"prettier.semi": false,
}
```

Otra extensión popular se [Better Comments](https://marketplace.visualstudio.com/items?itemName=aaron-bond.better-comments), que nos permite resaltar nuestros comentarios y clasificarlos en diferentes categorías, por ejemplo, rojo para los comentarios que indiquen algo que hay que cambiar, o verde para los que indiquen que se realizó un cambio correcto. Finalmente también utilizo una extensión llamada [Bracket Pair Colorizer](https://marketplace.visualstudio.com/items?itemName=CoenraadS.bracket-pair-colorizer-2), que nos ayuda a guiarnos a través del código cuando tenemos muchos paréntesis o llaves anidadas.

## Línea de Comandos

Una de las herramientas que más agilizan las tareas de un programador es la línea de comandos. Desde esta herramienta podemos tener acceso a casi todas las aplicaciones de nuestro sistema, e interactuar y dar instrucciones a nuestra computadora desde una sola aplicación. Podemos navegar entre directorios de una manera rápida, utilizar aplicaciones como Git, inicializar procesos como correr servidores para nuestras aplicaciones, utilizar editores de texto dentro de la misma terminal e incluso realizar peticiones HTTP con la ayuda de herramientas como cURL.

El beneficio principal de utilizar la línea de comando es la velocidad que ganamos al no depender de interfaces gráficas (GUI) para las aplicaciones que estamos corriendo. Por supuesto, no todas las apliaciones pueden ser utilizadas a través de la línea de comandos, como por ejemplo los navegadores como Chrome o Firefox, sin embargo, las aplicaciones que suelen utilizarse para programar usualmente tiene algún tipo de cliente que corra en la terminal, ya que es la herramienta preferida de una gran cantidad de desarrolladores.

Al utilizar las aplicaciones que disponen de versión en la línea de comandos también ganamos un mejor entendimiento de cómo funcionan dichas aplicaciones, ya que no estamos ligados a la manera en la que cada una de las aplicaciones deciden organizar sus interfaces, permitiendonos utilizar cualquier aplicación sin tener que adaptarnos estrictamente a la manera en la que esta funciona, y sin utilizar implementaciones o tecnologías propietarias, lo que nos proporciona mayor flexibilidad para decidir la manera en la que queremos trabajar. Asi por ejemplo, si en algún momento queremos cambiar de editor de código, vamos a seguir siendo capaces de compilar nuestros programas de la misma manera a través de la terminal, aunque diferentes editores tengan diferentes compiladores, o si utilizamos aplicaciones como Vim, vamos a poder utilizar un editor con el que estemos familiarizados incluso si estamos accediendo de manera remota a un servidor, etc.

La línea de comandos es uno de los puntos más importantes para aumentar nuestras posibilidades y reducir nuestros tiempos de desarrollo. Tareas que suelen tomar un tiempo considerable desde algun tipo de interfaz, como buscar un botón específico o utilizar funcionalidades que no entendemos cómo funcionan, pueden ser accedidas y ejecutadas con pocos comandos desde una terminal.

También tenemos diferentes opciones de customización para nuestras líneas de comandos, como la posibilidad de utilizar diferentes shells o diferentes terminal (la aplicación a través de la cuál accedemos a la línea de comandos de nuestra computadora). Sin embargo hasta cierto punto todas las líneas de comando son muy similares. En mi caso mi terminal de preferencia es [Alacritty](https://github.com/alacritty/alacritty) por su velocidad y customización. Utilizo [ZSH](https://ohmyz.sh/) como shell. Ocasionalmente utilizo Vim como editor de texto en terminal y [tmux](https://github.com/tmux/tmux/wiki) como terminal multiplexer.

## Vim

Así a como la línea de comandos la principal herramienta de muchos desarrolladores para aumentar su productividad al utlizar las múltiples aplicaciones de sus sistemas, mejorar la movilidad dentro de cada una de las aplicaciones que utilicemos también nos va a permitir mejorar nuestra experiencia de desarrollo, e incrementar nuestra productividad y velocidad.

Debido a que nuestro principal método de interacción con nuestras computadoras es nuestro teclado, saber cómo agilizar nuestra movilidad utilizando el teclado es una de las herramientas nos va a permitir. Herramientas como Vim nos proporcionan una manera sencilla y rápida de controlar nuestras aplicaciones (las que las keybinds de Vim) totalmente desde nuestro teclado, permitiéndonos ser mucho más rapidos a la hora de escribir y editar código, y navegar a través de la aplicación.

Los comandos de Vim estan diseñados de tal forma que el programador pueda unificar el proceso de analizar y escribir código en una misma acción. Al estar gran parte del tiempo dentro de un editor de código, solemos repetir gran cantidad de comandos como elimnar una línea, seleccionar una palabra, borrar un párrafo o navegar de arriba hacia abajo en el código.

Vim nos permite no depender el mouse para realizar este tipo de acciones, podemos por ejemplo, eliminar el contenido dentro de un par de paréntesis sin tener que ir a buscar con el cursor donde se cierran y donde se abren los paréntesis, o seleccionar etiquetas HTML que no sabemos donde terminan. Podemos borrar o reemplazar palabras sin necesidad de despegar nuestras manos del teclado, o copiar y pegar utilizando la posición natural de nuestros dedos en el teclado en vez de tener que buscar las teclas por defecto.

A pesar de que Vim puede ser difícil de aprender durante los primero 30 minutos de uso, al cabo de algunos días de práctica se logra superar la velocidad de edición previa al uso de Vim, y de ahí en adelante solo se va incrementando. Es similar a aprender touch-typing (escribir sin ver el teclado) en el sentido que no es necesario para desarrollar o escribir código, pero una vez se aprende, mejoramos mucho en comodidad y velocidad, dejamos de pensar en si estamos presionando la tecla correcta para simplemente pensar en lo que escribimos. El secreto para ser eficientes con Vim y realmente notar sus ventajas está en combinar los diferentes comandos, que si bien ya nos ahorran tiempo al ser utilizados de manera individual, pueden ser combinados para hacernos mucho más rápidos.

## Atajos en VSCode

La manera en la que combino todas estas herramientas es a través de las capacidades de customización que nos brinda VSCode. Para utilizar este editor con los atajos de Vim basta con instalar una [extensión](https://marketplace.visualstudio.com/items?itemName=vscodevim.vim) desde la tienda. Esto nos permite mantener las ventajas de utilizar un editor fácil de usar como VSCode, pero utilizando una forma eficiente de aprovechar las ventajas de este mismo al momento de escribir código.

Entre las otras opciones de customización que aplico a mi configuración de VSCode está el utilizar comandos similares a los que se utilizan en Vim para movilizarme entre los diferentes paneles de VSCode. Debido a que este editor está diseñado para cualquier tipo de programador, desde principantes hasta expertos, la manera por defecto de interactuar y movilizarse por el es a través del mouse. Así por ejemplo, para navegar entre nuestros archivos, renombrar, eliminar, crear paneles, dividir la pantalla (entre otras acciones comunes) dependemos nuevamente del mouse. Sin embargo, este editor nos proporciona otra API (similar a la que utilizamos para cambiar el tema) con la que podemos acceder a diferentes comandos de VSCode y de esta forma asignar keybinds o atajos en el teclado a cada una de estas acciones.
