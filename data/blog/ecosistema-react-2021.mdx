En los últimos años React se ha convertido en la tecnología más popular para el desarrollo de aplicaciones web modernas. Su popularidad se debe a múltiples factores como su facilidad de aprender, escalabilidad y mantenibilidad, que han hecho de React una librería capaz de ser utilizada tanto para proyectos pequeños y estáticos, como en aplicaciones más demandantes como es el caso de Facebook, Instagram, Twitter o Netflix.

Detrás de este éxito se haya también una fuerte e inmensa comunidad de desarrolladores de todas partes del mundo, y una igualmente grande colección de paquetes o librerías diseñadas para utilizarse de manera cómoda con React. Este ecosistema que ha nacido al rededor de React ha madurado lo suficiente como para ser considerado parte fundamental de la experiencia de desarrollo y hacen que cada día más personas empiezen a utilizar y recomendar esta librería.

## Qué es React

React es una librería de JavaScript utilizada para el desarrollo de interfaces y aplicaciones web mediante bloques de código independientes y reutilizables, llamados "componentes". Estos administran su propio estado y retornan JSX, un tipo de sintáxis basada en XML que nos permite incorporar funcionalidad de manera directa al elemento HTML al que representa cada componente. Por ejemplo:

```jsx
const element = <h1>Component</h1>
```

Esta sintáxis permite relacionar estríctamente el estado de cada componente con la estructura del mismo, asociando la lógica con la manera en la que este elemento se va a mostrar en la aplicación. Una de las características que ponen a JSX por encima de HTML común, es su capacidad de incorporar JavaScript de manera directa en cada componente, permitiéndonos controlar sus características o información en base a cierto estado.

Dicho estado puede ser implementado en cualquier componente, y almacenar cualquier valor que queramos que cada elemento administre por sí mismo. Así el estado de algún componente podría almacenar información sobre cuántas veces se hizo click sobre un botón, indicar que acciones se producen en respuesta a cualquier tipo de interacción por parte del usuario, o incluso suministrar un estado global a toda la aplicación, lo que nos permite implementar funcionalidad como autorización o un sistema de formularios, por nombrar algunos ejemplos.
